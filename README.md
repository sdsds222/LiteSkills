# 实验项目 LiteSkills：一种上下文压缩机制的Skills框架

**实验项目尚未完善，请谨慎使用。目前只实验过用Openclaw的openAI接口方式连接Gemini2.5pro，其他的模型可能会不兼容**

先看流程效果演示视频：https://www.bilibili.com/video/BV16Nc6zNEk2

一种针对于优化Skills缓解上下文过长臃肿问题的框架协议，将大量冗余多轮工具调用输入输出文本清除裁剪，实现节省token。

Skill开发者需要遵循规范格式，SKILL写法可以参考skills文件夹里面的SKILL.md，只有在SKILL.md中引入**<SKILL_CLEAN:Skill名称ID>**标签，才能够被模型识别为LiteSkill技能，即用于声明当前skill使用LiteSkill上下文回收框架协议

注意：开发者需要在Skill文档中指定需要总结保存的内容，模型会自动分析，为总结内容上方加上 **</SKILL_CLEAN:Skill名称ID>** 截止标签，进行除总结外的上下文垃圾回收裁剪。

执行上下文回收操作后，从Skill开始执行到结束之间的执行Skill的所有tool返回和assistant推理过程全部被裁剪回收，脚本会将 **<SKILL_CLEAN:Skill名称ID>** 和 **</SKILL_CLEAN:Skill名称ID>** 之间的冗余信息（包括Skill描述信息）清空。

只保留对Skill执行过程的浓缩的结论，实现节省上下文

裁剪掉一些skills的冗余信息可以一定程度上缓解注意力稀释，维持上下文纯净。

这个项目的想法主要还是想让不同skill的上下文的压缩逻辑不同，让skill开发者可以自定义最终skill保存在上下文中的总结信息。
